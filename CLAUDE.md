## Overview

This is a static personal portfolio site for kissmenerdygirl (DJ / Beatmaker based in Japan), hosted on GitHub Pages. No build step or framework — everything is plain HTML, CSS, and vanilla JS.

## Architecture

The site has three pages:
- `/index.html` — home page with links and latest note.com article
- `/works/index.html` — discography/works grid rendered from JSON
- `/events/index.html` — events list grouped by year rendered from JSON

All data is stored in `/assets/data/` as JSON files and fetched client-side at runtime:
- `works.json` — array of work items (title, artist, role, release_date, label, embed)
- `events.json` — array of events (type, title, date, location, description, media, links)
- `note_latest.json` — latest note.com article `{title, link, pubDate}`, auto-generated by CI

JS files in `/assets/js/`:
- `common.js` — runs on all pages: assembles mailto links from `data-user/domain/tld` attributes, sets footer year
- `works.js` — fetches `works.json`, renders cards sorted by `release_date` descending
- `events.js` — fetches `events.json`, splits into `hosted`/`appearance`, renders year-accordion with embedded X/Instagram/image media
- `note-latest.js` — fetches `note_latest.json`, populates the latest article link on the home page

## Data Formats

### works.json entry
```json
{
  "title": "...",
  "artist": "...",
  "role": "Remixer | Composer/Producer",
  "release_date": "YYYY-MM-DD",
  "label": "...",
  "isrc": "optional",
  "embed": { "type": "youtube | spotify", "src": "embed URL" }
}
```

### events.json entry
```json
{
  "type": "hosted | appearance",
  "title": "...",
  "date": "YYYY-MM-DD",
  "location": "...",
  "description": "...",
  "media": [
    { "type": "image", "src": "path", "alt": "...", "caption": "..." },
    { "type": "x", "url": "tweet URL" },
    { "type": "instagram", "url": "post URL" }
  ],
  "links": [{ "label": "...", "url": "..." }]
}
```

## GitHub Actions Workflows

- **`update-note-latest.yml`** — runs daily (12:00 JST), fetches RSS from note.com via `scripts/fetch_note.cjs`, writes `assets/data/note_latest.json` and commits
- **`append-event.yml`** — triggered when OWNER opens a GitHub Issue with the `event` label; parses the issue form body and appends to `events.json`, then closes the issue
- **`minify-images.yml`** — triggered on push to main when images change in `assets/img/`; runs `scripts/minify-changed.mjs` (imagemin) and commits back with `[skip ci]`

## Scripts

- `node scripts/fetch_note.cjs` — fetch latest note.com RSS and write `assets/data/note_latest.json` (requires `rss-parser`: `npm install rss-parser`)
- `node scripts/minify-changed.mjs <file1> <file2> ...` — minify specific image files in-place (requires imagemin packages: `npm install`)

## Content Updates

- **Add a work**: edit `assets/data/works.json` directly
- **Add an event**: open a GitHub Issue using the event template (OWNER only, triggers `append-event.yml`), or edit `assets/data/events.json` directly — keep sorted by `date` descending
- **Add images**: place in `assets/img/`, push to main — minification runs automatically via CI
